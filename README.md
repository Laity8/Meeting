# Meeting
大三阶段项目--视频会议
## 客户端

- **工具**：使用QT5.12框架开发图形页面
- **构建环境**：MinGW （GCC ，GDB）
- **功能**：登录注册，修改个人信息，创建加入会议，桌面共享，视音频，滤镜萌拍。
- **数据传输**：使用Windows套接字进行数据传输，服务器与客户端统一自定义协议，使用结构体进行数据封装，必要时对数据进行序列化。
- **网络库设计**：将TCP传输和UDP传输抽象成一个网络类，同时创建网络中介者类来作接口，中介者与网络类为横向依赖关系。连接与数据处理使用阻塞多线程进行分离操作。
    - **优点**：解除了网络库实现与调用间的耦合，并且降低了文件间的依存关系。
- **视音频部分**
    - **音频**：使用QT音频采集功能进行数据采集，之后使用speex库对数据进行编码（压缩），在进行发送。收到数据后在解码（解压缩）
        - **静音检测**：调用webrtc库对捕捉的数据进行比对。
        - 
    - **视频**：对于摄像头，使用opencv库的capture对画面进行捕捉（因opencv库功能强大，方便后面萌拍功能的人脸识别）；对于桌面采集，使用QT的桌面采集功能，使用QT库函数进行图片压缩操作。
- **遇到的问题**：
    - GUI卡顿：
        - 原因：QT事件循环中事件触发后处理过于频繁
        - 解决：用子线程对事件响应进行处理
    - 音频视频同时开启时卡顿：
        - 原因：视频数据过大导致TCP滑动窗口阻塞
        - 解决：为视频音频单独建立TCP传输通道
    - 音画不同步：
        - 原因：音频于视频数据发送的数据大小不同，或对端处理数据能力有限导致的视频数据阻塞
        - 解决：在数据包中添加时间戳，收到信息后根当前时间进行比对，差值过大可直接删除

### 服务器

- **工具**：Linux平台，C/C++，CLion IDE
- **构建环境：GNU**（GCC，GDB等）
- **数据库设计**：使用Mysql8.0.1，对mysql提供的C++ API进行封装并使用
- **数据传输**：使用Linux套接字进行数据传输，服务器与客户端统一自定义协议，使用结构体进行数据封装，必要时对数据进行反序列化。
- **网络库设计：**与客户端相同。
- **服务器架构：**使用Epoll LT同步阻塞多线程模型，监测epollin|epolloneshot事件，生产者消费者线程池模型
    - **epolloneshot优点**：保证只有一个线程服务socket，避免多线程在读取一个socket缓冲区时发生竞争导致错误*
- **改进方向**：音视频编码使用ffmpeg，网络库借鉴muduo或libevent，数据传输反射机制，心跳机制，日志系统
